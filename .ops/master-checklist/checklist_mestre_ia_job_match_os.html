Checklist Mestre IA — Job Match OS / Career CRM

Checklist operacional em HTML para execução e validação das fases do projeto. Cada item possui pré-validação obrigatória, evidência, status de execução e status de validação. Quem executa não valida. Quem valida não executa.

Documento impresso/exportado do Checklist Mestre IA.

Regra de Separação

A IA que executar um item não pode validar esse mesmo item. A IA que validar não pode executar esse mesmo item.
  • Executor marca apenas Pré-check e Executado.
  • Validador marca apenas Validado.
  • O nome/ID do executor deve ser diferente do validador.
  • Validação exige evidência mínima e status objetivo.

Antes de Executar

Antes de executar qualquer item, a IA deve verificar se há algo que precisa ser validado antes.
  • Consultar fases anteriores e relatórios .ops.
  • Verificar dependências do item.
  • Confirmar git status --short.
  • Confirmar se não há mock/placeholder/fake success.
  • Confirmar se a ação externa depende de API oficial ou aprovação humana.

Regra Anti-Fake

Tudo que estiver dentro do controle da plataforma deve ser funcional de verdade desde a Fase 0.
  • Não aceitar mock como entrega final.
  • Não aceitar simulação apresentada como funcional.
  • Não fingir candidatura, envio, integração ou sucesso.
  • Integração externa não configurada deve retornar estado honesto.

Critérios de aceite

1. Todos os itens executados têm evidência.
2. Validador diferente do executor.
3. Sem mocks como runtime principal.
4. Sem sucesso falso.
5. Sem ação externa sem aprovação.
6. Documentos .ops atualizados.
7. pnpm validate OK.
8. git status limpo.
9. Commit claro.
10. Push feito ou bloqueio documentado.
